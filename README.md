# 社团管理系统 (Community Management System)

## 1. 项目背景

随着高校社团活动的日益增多和社团规模的不断扩大，传统的社团管理方式（如手动记账、纸质审批、口头通知）已难以满足高效、透明、便捷的管理需求。信息不对称、流程繁琐、数据统计困难等问题日益凸显，影响了社团的健康发展和成员的活跃度。

为了解决这些痛点，我们开发了此社团管理系统。本系统旨在为高校社团提供一个数字化、一体化的管理平台，通过信息化手段简化管理流程，加强社团内部成员的沟通与协作，提升社团运营效率和成员的参与感。

## 2. 技术栈

本项目采用前后端分离的架构，具体技术选型如下：

* **后端技术栈:**
  
  * **核心框架:** SSM (Spring Boot + SpringMVC + MyBatis)
    * **SpringBoot:** 作为项目的核心容器，负责Bean的生命周期管理和依赖注入 (DI)。
    * **SpringMVC:** 负责处理Web请求，实现RESTful API接口，是前后端数据交互的桥梁。
    * **MyBatis:** 作为持久层框架，负责与数据库进行交互，实现对社团数据的增删改查 (CRUD)。
  * **数据库:** MySQL 8.0
  * **服务器:** Apache Tomcat
  * **构建工具:** Maven

* **前端技术栈:**
  
  * **核心框架:** Vue.js
  * **UI 框架:** Element UI
  * **路由管理:** Vue Router
  * **状态管理:** Vuex
  * **HTTP客户端:** Axios
  * **构建工具:** Webpack

## 3. 系统功能模块

系统主要包含以下几个核心功能模块：

1. **用户管理:**
   
   * **注册与登录:** 提供统一的登录入口，支持权限验证。
   * **个人信息管理:** 允许用户查看和修改个人资料。
   * **权限控制:** 区分普通成员、社团管理员、系统管理员等不同角色，分配不同操作权限。

2. **社团管理:**
   
   * **社团信息展示:** 集中展示所有社团的基本信息、简介、成员列表等。
   * **加入/退出社团:** 成员可以在线申请加入或退出社团，由管理员进行审批。
   * **社团创建与审批:** 支持在线创建新社团，并由系统管理员审核。

3. **活动管理:**
   
   * **活动发布:** 社团管理员可以发布新的社团活动，包括活动时间、地点、内容等。
   * **活动报名:** 社团成员可以在线浏览并报名参加感兴趣的活动。
   * **活动签到:** 提供活动签到功能，记录成员参与情况。
   * **活动历史归档:** 自动归档已结束的活动，方便回顾。

4. **新闻公告:**
   
   * **公告发布:** 管理员可以发布面向全校或特定社团的公告。
   * **新闻动态:** 展示社团的最新新闻和动态，促进信息流通。

5. **财务管理:**
   
   * **会费管理:** 记录社团成员的会费缴纳情况。
   * **账目明细:** 公开透明地记录社团的每一笔收入和支出，方便成员监督。

6.  **留言与交流:**
    *   **私信沟通:** 允许普通成员向社长或管理员发送私信，进行点对点的交流。
    *   **公共留言板:** 提供社团内部的公共留言区，方便成员进行公开讨论。

7.  **文件管理:**
    *   **文档上传与下载:** 管理员可以上传学习资料、社团规章、活动照片等文件，成员可按权限进行下载和浏览。
    *   **文件分类管理:** 支持对上传的文件进行分类归档，方便查找。

## 4. 特色与创新点

1.  **前后端分离架构:**
    采用SSM + Vue.js的前后端分离模式，使得开发职责更清晰，后端专注于业务逻辑和数据服务，前端专注于用户界面和交互体验。这种架构不仅提高了开发效率，也为未来的技术升级和移动端扩展（如开发小程序、App）打下了坚实基础。

2.  **精细化的权限管理:**
    系统内置了基于角色的访问控制（RBAC）模型，可以灵活地为不同用户（如系统管理员、社团社长、普通成员）分配不同的操作权限，确保了系统的安全性和管理的有序性。

3.  **数据驱动的社团运营:**
    系统能够自动记录和统计社团成员的活动参与度、社团的财务状况等关键数据。社团管理者可以基于这些数据进行分析，从而做出更科学的决策，例如调整活动策略、评估社团活跃度等。

4.  **一站式信息平台:**
    将社团信息、活动通知、新闻公告、财务公开等功能整合于一体，打破了信息孤岛，为社团成员提供了一个集中获取信息的统一入口，提升了沟通效率和用户体验。

5.  **集成AI模块 (Powered by QWen):**
    *   **智能问答助手:** 内置一个基于 **QWen** 大语言模型的智能问答机器人。用户可以用自然语言提问关于社团介绍、活动安排、入会流程等常见问题，机器人可提供7x24小时的实时解答，减轻管理员的重复工作。
    *   **活动内容智能生成:** 管理员在发布活动时，只需输入活动主题和要点，AI助手可以辅助生成详细、生动的活动描述和宣传文案。
    *   **智能推荐:** 基于用户的兴趣标签和历史活动参与记录，系统可以智能推荐用户可能感兴趣的社团或活动，提升用户的参与度和归属感。
## 5. 技术调整说明

1. **移除 Lombok 依赖**：为彻底规避旧版 javac 中 com.sun.tools 内部 API 触发的 `java.lang.NoSuchFieldError`，项目已删除 Lombok，改为手工维护 setter/getter、构造器等基础代码，从根源消除对过时技术的依赖。
2. **统一为普通 POJO**：`com.cms.entity` 以及 `com.cms.common.Result` 下的所有数据模型均以标准 POJO 形式存在，便于调试、排错与后续扩展。
3. **配置 Maven 编译插件**：在 `pom.xml` 中新增 `maven-compiler-plugin`，显式设置 Java 8 `source/target`，并追加 `-Xlint:-options` 编译参数，以确保在构建阶段就屏蔽掉旧选项警告并保持编译器行为可控。
